### 문제점

#### 이탈 방지용 콜라이더 적용 위치

**내용**

- 스크립트에서의 코딩 용이성을 위해 플레이어의 자식 오브젝트로 설계하려 했으나, 되려 플레이어를 따라가게 되고 스크립트가 더 길어지는 등의 문제 발생. 추가적으로 플랫폼에 해당 콜라이더 벽이 걸려 오히려 플레이어의 이동이 제한되는 현상 발생

**해결**

- 콜라이더 벽을 플레이어의 자식 항목으로 두지 않고 분리된 오브젝트로 설계 변경하면서, 오직 플레이어의 현재 y위치만을 갱신하도록 설정. 이전보다 오히려 변경된 상태가 관리하기 더 용이했음. 두 번째 문제는 콜리젼 그리드에서 콜라이더 벽과 오직 플레이어만이 충돌하도록 설정하려 했으나, 여러 문제들이 동시에 해결됨

### 알게된 내용

#### 확률 정의

**내용**

- 생성되는 플랫폼의 구멍 발현 확률을 구현하기 위해 여러 방법을 시도하던 중 로그함수를 고려하였는데, 최대값을 고정하기가 여의치 않아 폐기하였다.
- 다음으로는 1-Mathf.Pow(0.99,yPos)를 고려해 보았다. 캡을 설정하기도 좋고, 올라갈수록 구멍이 발현될 확률이 높아지는 것 까지 의도하였다. 다만, 증가되는 폭을 조절하기가 여의치 않았다.
- 마지막으로 적용한 내용으로, 유니티 공식 문서에서 발췌한 방법이다. 확률을 0.0~1.0 사이의 float 값으로 두지 않고 전체에서의 비율을 계산하는 방식이다. 
```csharp
float[] probs = {normalProbs, holeProbs};
        float total = 0;

        foreach (float a in probs)
        {
            total += a;
        }

        float randomPoint = Random.value * total;

        for(int i = 0;i< probs.Length; i++)
        {
            if(randomPoint < probs[i])
            {
                return (PlatformGenerator.PieceType)i;
            }
            else
            {
                randomPoint -= probs[i];
            }
        }

        return (PlatformGenerator.PieceType)probs.Length - 1;
```
여기서 `normalProbs`는 일반 플랫폼 조각이 나오도록 하는 가중치이고, `holeProbs`는 플레이어가 도달한 최고 높이로서 구멍 조각이 나오도록 하는 가중치이다. 이 두 수를 더해 전체 파이를 만들고, 랜덤으로 뽑힌 수치를 빼가면서 랜덤값이 어느 영역에 위치하는지를 정하여 확률을 구현한다.
